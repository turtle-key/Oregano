Sign in with Apple setup

  1. Go to Apple Developer > Identifiers and create a Service ID for Sign in with Apple.
  2. Enable Sign In with Apple on that Service ID, then configure:
    • Primary App ID (your app’s identifier)
    • Web Domain (your shop domain)
    • Return URLs (add the module redirect URI shown on the sociallogin module page)
  3. Create a Key for Sign in with Apple and download the .p8 private key (save it securely).
  4. Copy into the module:
    • Service ID (client_id)
    • Team ID
    • Key ID
    • Private Key (paste the full .p8 contents)
Apple requires a public HTTPS domain. For local testing, use a tunneling URL (e.g. ngrok) and add it as a return URL.

Privacy note: Users may choose “Hide My Email”. The module will create an account with the private relay address Apple returns.
